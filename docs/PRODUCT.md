# Legacy Vault - 产品设计文档

## 1. 产品概述

Legacy Vault 是一个军事级加密的数字保险箱应用。用户可以存储一生中最私密的信息——密码、遗嘱、秘密、私人信件、给后人的话——并设定精确的继承规则，确保在正确的时刻将正确的内容交给正确的人。

## 2. 用户画像

### 主要用户
- **家庭责任者** (30-60岁)：有配偶/子女，希望做好万一的准备
- **隐私敏感者** (所有年龄)：需要绝对安全的存储空间
- **数字资产持有者**：加密货币钱包、重要账号密码

### 次要用户
- **企业主**：公司关键信息的安全传承
- **老年人**：数字化遗嘱和身后安排
- **旅行者/高危职业**：出发前做好信息安排

## 3. 核心用户旅程

### 旅程A：日常使用（密码管理器）
```
用户登录 → 浏览/搜索密码 → 自动填充 → 新增/修改密码
```

### 旅程B：存储私密内容
```
打开保险箱 → 选择类型(文字/文件/媒体) → 加密存储
→ 设置继承人(可选) → 设置触发条件(可选)
```

### 旅程C：设置遗产计划
```
进入遗产规划 → 添加继承人 → 分配内容
→ 设置触发条件 → 设置验证方式 → 确认激活
```

### 旅程D：继承人接收
```
触发条件满足 → 系统通知继承人 → 身份验证
→ 多人确认(如设置) → 解锁对应内容
```

## 5. 安全架构

### 加密方案
```
用户主密码
    ↓ (PBKDF2/Argon2 派生)
主密钥 (Master Key)
    ↓
  ┌─────────────────┐
  │ 数据加密密钥     │ ← AES-256-GCM 加密每条数据
  │ (每条数据独立)   │
  └─────────────────┘
    ↓
  ┌─────────────────┐
  │ 继承人密钥分片   │ ← Shamir秘密共享
  │ (M-of-N 阈值)   │    需要M个继承人中的N个确认
  └─────────────────┘
```

### 零知识架构
- 服务端只存储加密后的数据
- 解密密钥永远不离开用户设备
- 我们无法读取用户任何数据
- 即使数据库泄露，数据也是安全的

### 生存验证协议
```
定期检查(可配置) → 用户响应 → 重置计时器
                  ↓ (未响应)
              发送提醒(多渠道)
                  ↓ (仍未响应)
              通知紧急联系人确认
                  ↓ (确认用户已故)
              启动继承协议
```

## 7. 技术选型（推荐）

| 层 | 技术 | 原因 |
|----|------|------|
| 前端 | React + TypeScript | 安全审计工具丰富 |
| 移动端 | React Native | 访问安全存储API |
| 加密 | libsodium (NaCl) | 经过验证的加密库 |
| 密钥管理 | Web Crypto API | 浏览器原生安全 |
| 数据库 | SQLCipher (本地) | 加密SQLite |
| 云存储 | S3 + 客户端加密 | 零知识云存储 |
| 认证 | WebAuthn / FIDO2 | 无密码认证 |
| 后端 | Rust / Go | 安全性能关键服务 |
