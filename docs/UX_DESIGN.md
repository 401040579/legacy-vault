# Legacy Vault - 用户体验设计文档

> 安全产品的UX是巨大挑战：如何让军事级加密对用户透明？如何让遗产规划不显得沉重？如何在安全与便利之间找到平衡？

---

## 1. 设计理念

### 1.1 核心设计原则

| 原则 | 说明 | 实践 |
|------|------|------|
| **温暖的安全感** | 安全不是冰冷的技术术语，而是一种温暖的守护感 | 用"守护"替代"加密"，用"传递"替代"分发" |
| **隐形加密** | 加密过程对用户完全透明，不需要理解密码学 | 自动加密/解密，用户只感受到"写入"和"打开" |
| **渐进式复杂度** | 从简单开始，需要时再展示高级选项 | 基础流程3步完成，高级设置折叠隐藏 |
| **生命而非死亡** | 产品围绕"守护生命中重要的东西"，而非"为死亡做准备" | 品牌语言强调"传承"、"守护"、"留下" |
| **信任可视化** | 让用户看到安全在发生 | 加密动画、安全指示器、零知识证明演示 |

### 1.2 品牌语言指南

**使用**：
- 守护、传承、留下、珍藏、安心、信赖、保管
- "你的数字保险箱"、"安全传递给你爱的人"
- "只有你能打开"、"连我们也看不到"

**避免**：
- 死亡、去世、身后事、遗嘱（除法律模板外）
- 技术术语：加密算法、密钥派生、哈希函数
- 恐惧营销："万一出事"、"你的数据可能泄露"

**中性化表达映射**：
| 技术/法律概念 | 用户界面用语 |
|--------------|------------|
| 端到端加密 | "只有你能打开" |
| 零知识架构 | "连我们也看不到你的内容" |
| 继承人 | "守护人" / "接收人" |
| 触发条件 | "传递时机" |
| 死亡触发 | "当你无法亲自传递时" |
| 生存验证 | "安心确认" / "定期问候" |
| 紧急访问 | "紧急传递" |
| Shamir分片 | "安全碎片"（仅在高级设置中出现） |
| 密钥派生 | 不出现 |
| 恢复密钥 | "紧急恢复词" |

## 2. 情感化设计

### 2.1 情感设计框架

Legacy Vault涉及用户最深层的情感——对家人的爱、对死亡的恐惧、对隐私的渴望。设计必须在每个环节考虑用户的情感状态。

**用户情感旅程**：

```
首次接触        注册           日常使用         设置遗产计划      触发/接收
  |               |              |                |                |
  v               v              v                v                v
好奇 + 犹豫    谨慎 + 期待    习惯 + 安心     沉重 + 释然      悲伤 + 感恩
  |               |              |                |                |
  需要：          需要：         需要：           需要：           需要：
  信任建立        安全引导      便捷无感         情感支持         温暖呈现
  价值展示        简单上手      安全可见         步骤清晰         尊重隐私
```

### 2.2 关键情感设计点

#### 注册阶段——"安全感的第一印象"
- 动画展示"你的数据如何被保护"（信封→锁→保险箱的隐喻）
- 主密码创建时不仅显示强度，还显示"即使是我们的工程师也无法看到你的密码"
- 恢复密钥生成时：用"这是你唯一的备用钥匙，请像保管家门钥匙一样保管它"代替"请保存恢复密钥"

#### 遗产规划阶段——"化解沉重感"
- 不直接问"你去世后想把东西给谁"，而是"如果有一天你无法亲自交给他们，你希望谁来守护这些？"
- 规划完成后显示：
  - "你已经为最爱的人做好了安排 ♡"
  - "XX条内容已安全托付，你可以安心了"
- 提供"写一封信给他们"的入口，将遗产规划转化为情感表达

#### 内容接收阶段——"温暖的传递"
- 继承人打开内容时的界面设计应该像拆开一封珍贵的信，而非打开一个冰冷的文件管理器
- 显示原始用户的名字和头像（如果设置了）
- 如果有"给你的一句话"（用户设置的寄语），优先展示
- 视频/录音自动创建温馨的播放界面

### 2.3 色彩与视觉语言

| 元素 | 设计选择 | 情感含义 |
|------|---------|---------|
| 主色调 | 深靛蓝 (#1E3A5F) | 信任、安全、深邃 |
| 辅色调 | 暖金色 (#D4A853) | 珍贵、传承、温暖 |
| 安全指示 | 翡翠绿 (#2ECC71) | 安全、保护、生机 |
| 警告/注意 | 琥珀色 (#F39C12) | 注意、温和的警示 |
| 危险/错误 | 玫瑰红 (#E74C3C) | 紧急但不恐吓 |
| 背景 | 米白/暖灰 | 温暖、家的感觉 |
| 字体 | 衬线体（标题）+ 无衬线体（正文） | 传承感 + 现代感 |
| 图标风格 | 圆润线条，微笑曲线 | 友好、可信赖 |
| 插画风格 | 手绘温暖插画（非矢量冷色调） | 人性化、有温度 |

### 3.2 注册流程（5步，目标<5分钟）

```
步骤1：基本信息
  [邮箱输入框]
  [或] 社交登录（Google/Apple/微信）
  "我们只用邮箱发送重要通知，不会发送垃圾邮件"

步骤2：创建主密码
  [密码输入框] + 实时强度指示器
  [确认密码]
  提示气泡："这是打开你保险箱的唯一钥匙。请选择一个你记得住但别人猜不到的密码。"
  安全提示：建议12位以上，包含大小写、数字和符号
  底部小字："我们使用零知识加密，永远不会存储你的密码"

步骤3：保存紧急恢复词
  显示24个单词的恢复词
  [下载PDF] [打印] [手动抄写确认]
  提示："请务必保存这些恢复词！如果你忘记主密码，这是唯一的恢复方式。我们无法帮你重置密码。"
  确认复选框："我已安全保存恢复词，并理解丢失后数据将无法恢复"

步骤4：安全演示（可跳过）
  互动演示：用户输入一句话 → 看它被加密 → 看它被解密
  "看到了吗？你的数据在离开你的设备前就已经被加密了。"

步骤5：开始使用
  三个快速选项卡片：
  - "保存第一个密码" → 进入密码管理
  - "写一封信" → 进入安全笔记
  - "先随便看看" → 进入主页
```

## 4. 安全感的用户体验传达

### 4.1 加密过程可视化

用户每次保存内容时，显示一个简短（<1秒）的加密动画：
```
明文 → [锁定动画] → "已安全存储 ✓"
```
- 动画简洁优雅，不打断用户流程
- 让用户"感受"到加密在发生，但不需要理解技术细节
- 首次使用时可展示详细版本（可跳过）

### 4.2 安全状态指示器

在应用顶部或底部栏永久显示安全状态：
```
🛡️ 安全连接 · 零知识加密 · 最后同步: 2分钟前
```

### 4.3 "透明窗"设计

在设置页面提供"安全透明窗"，让好奇的用户可以查看：
- 当前使用的加密算法
- 密钥派生参数
- 数据存储位置
- 最近的安全审计报告

### 4.4 安全操作确认

关键操作（修改继承计划、删除内容、紧急访问）使用"安全确认模式"：
```
┌──────────────────────────────────────────┐
│ 🔐 安全确认                              │
│                                          │
│ 你正在修改遗产传承计划。                  │
│ 这是一个重要操作，请输入主密码确认。       │
│                                          │
│ [主密码输入框]                            │
│                                          │
│           [取消]  [确认修改]               │
└──────────────────────────────────────────┘
```

## 5. 无障碍设计 (Accessibility)

| 标准 | 要求 |
|------|------|
| WCAG 2.1 AA | 最低合规标准 |
| 屏幕阅读器 | 完全支持VoiceOver/TalkBack |
| 键盘导航 | 所有功能可通过键盘完成 |
| 色彩对比 | 所有文本对比度 >= 4.5:1 |
| 字体大小 | 支持系统级字体缩放 |
| 触控目标 | 最小44x44px |
| 动画 | 尊重"减少动态效果"系统设置 |
| 多语言 | 界面支持RTL布局（阿拉伯语等） |

### 老年用户特殊考虑（画像C）

- 提供"简洁模式"：更大字体、更少选项、更直接的流程
- 关键按钮使用文字而非图标
- 支持语音输入笔记
- 操作步骤附带简短视频教程

## 6. 响应式设计

### 断点策略

| 设备 | 断点 | 布局策略 |
|------|------|---------|
| 手机 | < 768px | 单列，底部导航栏，卡片式内容 |
| 平板 | 768px - 1024px | 双栏，侧边栏导航 |
| 桌面 | > 1024px | 三栏，完整侧边栏+内容+详情 |

### 交互差异

| 操作 | 手机 | 桌面 |
|------|------|------|
| 解锁 | 生物识别优先 | 主密码优先 |
| 密码复制 | 长按复制 | 点击图标复制 |
| 文件上传 | 相机/相册选择器 | 拖拽上传 |
| 导航 | 底部标签栏 | 左侧边栏 |

## 8. 动效设计

### 8.1 核心动效

| 场景 | 动效 | 时长 | 目的 |
|------|------|------|------|
| 应用解锁 | 锁→开的转场 | 300ms | 安全感仪式 |
| 数据加密 | 文字→密文渐变 | 500ms | 加密可视化 |
| 数据解密 | 密文→明文渐变 | 500ms | 解密可视化 |
| 保存成功 | 柔和的对勾弹出 | 200ms | 确认反馈 |
| 页面切换 | 平滑滑动 | 250ms | 流畅感 |
| 删除确认 | 轻微震动 | 100ms | 慎重操作提示 |
| 继承人连接 | 两个圆形节点连线 | 400ms | 信任关系可视化 |

### 8.2 动效原则

- 所有动效必须尊重系统"减少动态效果"设置
- 动效不应阻塞用户操作（非模态）
- 首次出现完整版，后续出现简化版
- 加密/解密动效可在设置中关闭
